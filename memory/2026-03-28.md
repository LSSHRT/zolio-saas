# 2026-03-28 — Session debug zolio-saas

## Contexte
- Val a signalé des erreurs sur zolio-saas après la session du 26/03
- Erreur principale : `fetchevent.respondwith received error: TypeError: load failed` sur `/api/prospection/campaign` (502)
- Le domaine zolio.site était suspendu par ICANN (vérification WHOIS) → résolu par Val chez Amen

## Fixes

### 1. Service Worker (sw.js)
- Le SW interceptait TOUS les appels fetch → causait des crashs sur erreurs API
- Fix : supprimé l'interception fetch, le SW ne gère plus que les push notifications
- Commits : `fc9f524`, `aad1ccb`

### 2. Config VAPID lazy (push-notifications.ts)
- `webpush.setVapidDetails()` s'exécutait au build time → crash si clés VAPID absentes
- Fix : config lazy via `ensureVapidConfigured()` — appelée seulement à l'utilisation

### 3. JSON.parse sécurisé
- `parsePhotos()` dans devis/route.ts et devis/[numero]/route.ts → ajout try-catch
- `parseGeneratedLines()` dans ai/generate-devis → ajout try-catch
- `safeJsonParse()` helper ajouté dans http.ts
- `public/devis/[numero]/route.ts` → utilise safeJsonParse

### 4. Rate limit amélioré
- `rateLimitResponse()` helper avec header `Retry-After`
- Routes devis, clients, public devis mises à jour

### 5. Tests
- 5 nouveaux tests pour safeJsonParse et rateLimitResponse
- Total : 118 tests (100% passent)
